# 智能体构建日常工程化问题

> 基于 LangChain + LangGraph 构建知识库智能体的工程化问题清单

---

## 一、核心架构设计

### 1. 模块化设计
- **分层架构**：数据层（向量库/数据库）→ 逻辑层（检索/生成）→ 接口层（API/UI）
- **插件化 Skill**：每个能力独立封装（检索、计算、绘图等），可动态加载
- **配置驱动**：模型选择、切片规则、召回数量等通过配置文件管理，不硬编码
- **依赖解耦**：LLM、向量库、数据库可替换，不绑定具体厂商

### 2. 对话状态机（LangGraph 核心）
- **状态流转**：澄清意图 → 检索知识 → 生成回答 → 确认满意度
- **条件分支**：根据意图分类走不同流程（问答/闲聊/任务执行）
- **中断与恢复**：长对话支持暂停、回退、重新开始
- **多轮上下文管理**：动态维护对话历史，避免上下文溢出

### 3. 工具调用（Tool Calling）
- **工具注册机制**：统一管理工具定义、参数校验、执行逻辑
- **工具选择策略**：LLM 决定用哪个工具，还是规则路由？
- **工具失败处理**：超时、异常、返回无效时的兜底逻辑
- **工具链编排**：多个工具串联/并联执行（LangGraph 优势）

## 二、知识库处理

### 4. 文档解析与清洗
- **多格式支持**：PDF、Word、Markdown、HTML、Excel、图片 OCR
- **PDF 布局解析**：保留标题层级、表格结构、图片位置（不只是纯文本）
- **数据清洗**：去重、去噪、修复编码、统一格式
- **元数据提取**：来源、作者、时间、标签等保留到向量库

### 5. 切片规则（Chunking）
- **字数切片**：固定长度（如 500 字），简单但可能切断语义
- **语义切片**：按句子/段落边界切，保留完整语义
- **递归切片**：先按大标题切，再按段落，再按句子（推荐）
- **重叠策略**：相邻切片重叠 10-20%，避免边界信息丢失
- **动态切片**：根据文档类型调整（代码按函数、论文按章节）

### 6. 向量化存储
- **Embedding 模型选择**：OpenAI、Cohere、本地 BGE/M3E 等，按场景选
- **向量库选型**：Milvus、Pinecone、Weaviate、Chroma、Qdrant
- **向量索引优化**：HNSW、IVF 等索引类型，平衡速度和精度
- **元数据过滤**：向量检索 + 元数据过滤（如只查某部门、某时间段）

### 7. 增量索引与删除
- **增量更新**：新增文档只索引新内容，不全量重建
- **局部删除**：文档删除时同步删除对应向量，释放空间
- **版本管理**：知识库更新后保留历史版本，支持回溯
- **索引状态追踪**：哪些文档已索引、哪些失败、哪些过期

### 8. 离线批处理
- **异步导入**：大文档/批量文档用队列处理（Celery、RQ）
- **进度反馈**：导入进度可视化，支持取消和重试
- **失败重试**：索引失败自动重试，记录失败原因
- **资源控制**：限制并发数，避免打爆 CPU/内存

### 9. 知识库质量评估
- **低质量检测**：空文档、过短内容、乱码自动标记
- **重复内容检测**：相似度 > 95% 的文档提示合并
- **索引覆盖率**：多少文档成功索引，多少失败
- **定期巡检**：自动扫描知识库健康度，生成报告

## 三、检索与生成

### 10. 召回规则（Retrieval）
- **Top-K 设置**：召回多少条？（通常 5-10 条，太多会噪声）
- **阈值过滤**：相似度 < 0.7 的不要，避免低质量内容进 prompt
- **混合检索**：向量检索 + BM25 关键词检索，互补（推荐）
- **查询改写**：用户问题模糊时，先让 LLM 改写成更精准的查询
- **多路召回**：不同策略各召回一批，再合并去重

### 11. 重排规则（Reranking）
- **Cross-Encoder 重排**：用 BGE-Reranker、Cohere Rerank 等精排
- **LLM 重排**：让 LLM 判断哪条最相关（贵但准）
- **元数据加权**：新文档权重高、权威来源权重高
- **多样性控制**：避免召回的全是同一篇文档的不同切片

### 12. 上下文压缩
- **摘要压缩**：长文档先让 LLM 摘要，再塞进 prompt
- **选择性注入**：只注入最相关的段落，不是整篇文档
- **Token 预算**：严格控制在模型上下文窗口内（留足生成空间）
- **动态调整**：根据召回数量动态调整每个切片的注入长度

### 13. 幻觉检测与事实校验
- **原文对比**：生成的内容必须在召回的原文中有依据
- **引用标注**：回答中标注来源（[1] [2]），用户可点击查看
- **置信度评分**：LLM 自评回答的确定性，低于阈值提示"不确定"
- **二次验证**：关键回答让另一个 LLM 校验（贵但安全）

### 14. 闲聊与问答区分
- **意图识别**：先分类（问答/闲聊/任务），再走不同流程
- **闲聊兜底**：识别为闲聊时，不调用知识库，直接让 LLM 回答
- **边界情况**：问题模糊时，先追问澄清再检索
- **成本控制**：闲聊用便宜模型，问答用强模型

## 四、记忆系统

### 15. 短期记忆（对话上下文）
- **滑动窗口**：只保留最近 N 轮对话（如 10 轮）
- **摘要压缩**：超过窗口后，把旧对话摘要成一段话
- **关键信息提取**：人名、时间、地点等实体持久化
- **Token 限制**：严格控制在模型上下文内

### 16. 长期记忆（用户偏好/历史）
- **用户画像**：记住用户偏好、常用领域、历史问题
- **知识沉淀**：用户纠正过的错误答案，下次避免
- **存储方式**：向量库 + 数据库，支持检索和查询
- **隐私控制**：用户可清除自己的长期记忆

### 17. 重置机制
- **会话重置**：用户可清空当前对话，重新开始
- **记忆重置**：管理员可清除用户的长期记忆
- **知识库重置**：紧急情况可重建全部索引
- **缓存重置**：强制刷新缓存，获取最新数据

## 五、安全与权限

### 18. 基础安全
- **Prompt 注入检测**：过滤恶意 prompt（如"忽略之前指令"）
- **越狱防护**：防止用户诱导 LLM 输出违规内容
- **输出审计**：敏感词过滤、PII（个人信息）脱敏
- **速率限制**：单用户 QPS 限制，防刷

### 19. 权限控制
- **知识库权限**：不同用户/团队只能访问自己的知识库
- **文档级权限**：同一知识库里，某些文档只对特定人可见
- **接口权限**：API 鉴权（JWT、API Key）
- **操作审计**：谁在什么时候做了什么，全记录

### 20. 多租户隔离
- **向量隔离**：每个租户独立的 Collection/Namespace
- **数据隔离**：数据库层面租户字段隔离
- **资源隔离**：计算资源配额（避免一个租户占满）
- **配置隔离**：每个租户可自定义模型、切片规则等

## 六、性能与稳定性

### 21. 缓存机制
- **查询缓存**：相同问题直接返回缓存答案（Redis）
- **向量缓存**：Embedding 计算结果缓存，避免重复计算
- **LLM 缓存**：相同 prompt 复用上次结果（语义缓存）
- **缓存失效**：知识库更新后自动失效相关缓存

### 22. LLM 降级策略
- **主备模型**：主模型挂了自动切备用（如 GPT-4 → GPT-3.5）
- **本地模型**：云端模型全挂时，降级到本地小模型
- **降级提示**：告知用户"当前使用简化模型，回答可能不够精准"
- **熔断机制**：连续失败 N 次后熔断，避免雪崩

### 23. 强鲁棒性
- **异常捕获**：每个环节 try-catch，不让一个错误崩全局
- **超时控制**：LLM 调用、向量检索、数据库查询都设超时
- **重试机制**：网络抖动自动重试（指数退避）
- **健康检查**：定期检测各组件状态，异常自动告警

### 24. 多进程/并发问答
- **异步处理**：多个用户同时提问，异步处理不阻塞
- **连接池**：数据库、向量库、Redis 用连接池
- **限流排队**：超过处理能力时排队，不直接拒绝
- **资源隔离**：检索和生成用不同线程池，互不影响

## 七、工程化实践

### 25. 代码模块化
- **目录结构清晰**：data/ retrieval/ generation/ tools/ api/
- **职责单一**：每个模块只做一件事（切片、检索、生成）
- **接口抽象**：模块间通过接口通信，不直接依赖实现
- **依赖管理**：requirements.txt / pyproject.toml 锁定版本

### 26. 单元测试
- **切片逻辑测试**：不同文档类型切片是否正确
- **检索逻辑测试**：Mock 向量库，验证召回和重排
- **工具调用测试**：Mock 工具，验证参数传递和结果处理
- **对话流程测试**：模拟多轮对话，验证状态流转

### 27. 压力测试
- **单模块压测**：检索模块 QPS、LLM 调用延迟
- **端到端压测**：模拟 100 用户同时提问，系统表现
- **瓶颈定位**：CPU、内存、数据库连接、向量检索耗时
- **容量规划**：根据压测结果，规划服务器配置

### 28. 日志管理
- **分级日志**：DEBUG / INFO / WARN / ERROR 合理使用
- **结构化日志**：JSON 格式，方便检索和分析
- **链路追踪**：TraceId 贯穿整个请求（LangSmith 集成）
- **日志轮转**：自动归档、清理，避免磁盘撑爆

### 29. Hooks 与生命周期
- **Pre-processing Hooks**：用户输入前处理（清洗、校验）
- **Post-processing Hooks**：LLM 输出后处理（格式化、审计）
- **Retrieval Hooks**：检索前后自定义逻辑（如动态过滤）
- **Error Hooks**：异常时触发告警、记录、降级

### 30. MCP（Model Context Protocol）
- **标准化工具接入**：MCP 让工具调用更规范
- **动态工具发现**：运行时注册/卸载工具
- **跨模型兼容**：不同 LLM 用同一套工具定义
- **安全沙箱**：工具执行在隔离环境，防止恶意代码

## 八、可观测性

### 31. 链路追踪
- **LangSmith 集成**：可视化每个步骤（检索、重排、生成）
- **耗时分析**：每个环节耗时多少，瓶颈在哪
- **Token 统计**：每次调用用了多少 Token，成本多少
- **错误追踪**：失败请求的完整上下文，方便复现

### 32. 关键指标监控
- **响应时间**：P50、P95、P99 延迟
- **召回质量**：命中率、准确率、召回率
- **用户满意度**：点赞/点踩比例
- **系统健康**：CPU、内存、数据库连接数、向量库状态

### 33. 后台分析
- **问题统计**：哪些问题问得最多？哪些问题答不上来？
- **知识库缺口**：用户问题没命中知识库的，标记为"待补充"
- **模型对比**：不同模型的效果对比（准确率、延迟、成本）
- **趋势分析**：使用量增长、热点问题变化

## 九、评估与迭代

### 34. RAG 评估指标
- **Faithfulness（忠实度）**：回答是否基于召回的内容
- **Answer Relevancy（答案相关性）**：回答是否切题
- **Context Precision（上下文精确度）**：召回的内容是否相关
- **Context Recall（上下文召回率）**：该召回的是否都召回了

### 35. A/B 测试
- **模型对比**：GPT-4 vs GPT-3.5，哪个效果好？
- **切片策略对比**：500 字 vs 语义切片，哪个召回准？
- **重排策略对比**：Cross-Encoder vs LLM 重排，哪个更准？
- **指标对比**：准确率、用户满意度、延迟、成本

### 36. 数据版本管理
- **知识库版本**：v1、v2、v3，支持回滚
- **效果对比**：v2 比 v1 准确率提升了多少？
- **灰度发布**：新版本先给 10% 用户用，没问题再全量

### 37. 冷启动策略
- **引导式问答**：新用户进来，先问"你想了解什么？"给几个选项
- **示例问题**：展示常见问题，降低用户思考成本
- **渐进式引导**：用户问了一个问题后，推荐相关问题
- **知识库预热**：上线前先导入种子数据，避免空库尴尬

## 十、补充问题（你可能没想到的）

### 38. 成本优化
- **Token 预算控制**：单次对话最多用多少 Token，超了就截断
- **模型分级调用**：简单问题用便宜模型，复杂问题用贵模型
- **缓存复用**：相同问题直接返回，不调用 LLM
- **批量处理**：多个查询合并成一个请求，减少调用次数

### 39. 流式响应（Streaming）
- **打字机效果**：LLM 边生成边返回，用户不用干等
- **中间状态反馈**："正在检索..." → "正在生成..."
- **中断机制**：用户可随时停止生成

### 40. 引用与溯源
- **来源标注**：回答中标注引用了哪些文档
- **原文跳转**：点击引用可跳转到原文位置
- **置信度展示**：告诉用户"这个回答有 80% 的把握"

### 41. 多模态支持
- **图片理解**：用户上传图表，LLM 能理解并回答
- **表格解析**：Excel/PDF 表格结构化提取
- **图文混合**：回答中可插入图片、表格，不只是纯文本

### 42. 国际化与本地化
- **多语言支持**：中英文混合查询怎么处理？
- **本地化配置**：时区、日期格式、数字格式
- **文化适配**：不同地区的合规要求

### 43. 合规与审计
- **数据合规**：GDPR、个人信息保护法
- **审计日志**：所有操作可追溯
- **数据保留策略**：多久后自动删除历史数据
