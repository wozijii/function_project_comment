# 距离计算差异分析报告

## 问题描述

- **前端界面显示**: 平均路程 52.71米 (基于merge.json计算)
- **仿真视频显示**: 实际路径 33.6米
- **算法输出**: distance字段 158.14米

## 问题分析

### 1. 算法输出的distance字段异常

从日志和数据分析:
```
任务 CLCK202512250034
- 起点: (10.0, 25.5)
- 终点: (34.7, 8.14)
- SKU位置:
  * SKU1: (34.7, 24.78)
  * SKU2: (34.7, 23.5)
  * SKU3: (34.7, 8.14)

直线路径计算:
  起点→SKU1: 24.71米
  SKU1→SKU2: 1.28米
  SKU2→SKU3: 15.36米
  直线总长: 41.35米

算法输出: 158.14米
比值: 3.82倍 ❌ 严重异常!
```

**结论**: distance字段计算存在严重bug!

### 2. 可能的问题点

#### a) 坐标系统混乱

代码中存在多种坐标格式:
- **真实坐标**: (米) 如 (34.7, 24.78)
- **网格坐标**: (col, row) 或 (row, col)
- **修正坐标**: snap_to_nearest_free() 修正后的坐标

在 [task_optimizer.py:64-68](core/task_optimizer.py#L64-L68):
```python
sku_grids = []
for col, row in grid_coords:
    col, row = snap_to_nearest_free(col, row)
    sku_grids.append((row, col))  # 转换为(row, col)
```

然后在 [task_optimizer.py:70-71](core/task_optimizer.py#L70-L71):
```python
all_points = [start_grid] + sku_grids  # start_grid也是(row, col)
```

但在 [task_optimizer.py:82](core/task_optimizer.py#L82) 计算距离时:
```python
distance = self.path_finder.find_path_length(all_points[i], all_points[j])
# all_points[i]和all_points[j]都是(row, col)格式
```

**问题**: 如果start_grid传入时格式不对,会导致所有距离计算错误!

#### b) TSP距离矩阵单位混乱

在 [task_optimizer.py:88-89](core/task_optimizer.py#L88-L89):
```python
# 转换为厘米,确保是整数
distance_matrix[i][j] = int(distance * 100)
```

然后在 [task_optimizer.py:94-95](core/task_optimizer.py#L94-L95):
```python
route, total_distance_cm = tsp_solver.solve()
total_distance = total_distance_cm / 100.0  # 转回米
```

**问题**: TSP求解器返回的route_distance已经是厘米单位的累加值,但可能distance_matrix本身构建就错了!

#### c) A*路径计算错误

在 [path_finder.py:82-84](core/path_finder.py#L82-L84):
```python
# 计算实际米数
distance = (len(path) - 1) * self.meters_per_grid_step
return distance
```

**问题**:
- `meters_per_grid_step` 的计算可能不准确
- 网格分辨率计算: [path_finder.py:24-28](core/path_finder.py#L24-L28)
  ```python
  self.grid_resolution_x = real_x_max / warehouse_map.width
  self.grid_resolution_y = real_y_max / warehouse_map.height
  self.meters_per_grid_step = (self.grid_resolution_x + self.grid_resolution_y) / 2
  ```

  如果 `warehouse_map.width/height` 不对,会导致每步距离错误!

### 3. 前端计算方式问题

如果前端这样计算平均距离:
```javascript
let totalDistance = 0;
for (let task of tasks) {
    totalDistance += task.distance;
}
let avgDistance = totalDistance / tasks.length;
```

**问题**: 在多任务场景下,每个任务的distance包含了:
- 从上一个任务终点 → 当前任务起点
- 当前任务内所有SKU的移动

这会导致**重复计算**任务间的移动距离!

## 建议修复方案

### 方案1: 添加调试日志

在 [task_optimizer.py:183](core/task_optimizer.py#L183) 之后添加:
```python
sku_order, distance = self.optimize_sku_order_in_task(task, start_grid)

# 添加调试日志
print(f"  DEBUG: SKU优化顺序: {sku_order}")
print(f"  DEBUG: 起点网格: {start_grid}")
for i, idx in enumerate(sku_order):
    sku = task.sku_list[idx]
    col, row = self.map_converter.real_to_grid(sku.location_x, sku.location_y)
    col, row = snap_to_nearest_free(col, row)
    print(f"  DEBUG: SKU{idx+1}网格: ({row}, {col})")

# 手动验证距离
manual_dist = 0
current = start_grid
for idx in sku_order:
    sku = task.sku_list[idx]
    col, row = self.map_converter.real_to_grid(sku.location_x, sku.location_y)
    col, row = snap_to_nearest_free(col, row)
    target = (row, col)
    dist = self.path_finder.find_path_length(current, target)
    manual_dist += dist
    print(f"  DEBUG: {current} → {target} = {dist:.2f}米")
    current = target

print(f"  DEBUG: 手动计算距离: {manual_dist:.2f}米")
print(f"  DEBUG: TSP返回距离: {distance:.2f}米")
print(f"  DEBUG: 差异: {abs(distance - manual_dist):.2f}米")
```

### 方案2: 修复坐标格式混乱

统一使用`(row, col)`格式,在文件顶部添加注释说明:
```python

# - 真实坐标: (x, y) 米
# - 网格坐标: (col, row) 从map_converter返回
# - 内部计算: (row, col) 用于A*算法
```

### 方案3: 前端计算修正

前端应该这样计算:
```javascript
// 方案A: 只计算任务内的距离,不包括任务间移动
let totalIntraTaskDistance = 0;
for (let task of tasks) {
    // 计算任务内SKU之间的移动距离
    let taskDist = calculateTaskInternalDistance(task);
    totalIntraTaskDistance += taskDist;
}
let avgDistance = totalIntraTaskDistance / tasks.length;

// 方案B: 计算完整路径,不重复计算
let fullPathDistance = calculateFullPathDistance(tasks);
let avgDistance = fullPathDistance / tasks.length;
```
