# 路径问题修复说明

## 问题描述

之前的 `view/deal_nice_data.py` 使用相对路径 `../data/latest_input.json`,导致从不同目录运行时找不到文件,报错:

```
警告: 无法读取 simulate_id, 将使用默认路径: [Errno 2] No such file or directory: '../data/latest_input.json'
使用默认数据目录: ../data
保存汇总指标失败: [Errno 2] No such file or directory: '../data/workers.json'
```

## 解决方案

修改 `view/deal_nice_data.py`,使用绝对路径而不是相对路径:

### 修改前 (错误的相对路径)

```python
# 使用相对路径,依赖当前工作目录
with open('../data/latest_input.json', 'r', encoding='utf-8') as f:
    latest_input = json.load(f)

data_dir = Path(f"../data/data_{simulate_id}")
```

### 修改后 (正确的绝对路径)

```python
# 获取项目根目录(无论从哪里运行都能正确找到)
current_file = Path(__file__).resolve()
# view 目录的父目录就是项目根目录
project_root = current_file.parent.parent

# 使用绝对路径
latest_input_path = project_root / 'data' / 'latest_input.json'
with open(latest_input_path, 'r', encoding='utf-8') as f:
    latest_input = json.load(f)

data_dir = project_root / 'data' / f"data_{simulate_id}"
```

## 修复原理

使用 `Path(__file__).resolve()` 获取当前文件的绝对路径,然后通过 `.parent.parent` 获取项目根目录,再基于项目根目录构建所有其他路径。这样无论从哪个目录运行代码,都能正确找到文件。
